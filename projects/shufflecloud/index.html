How To Use ShuffleCloud
  • What's wrong with SoundCloud's shuffle?
      • It randomizes only the songs that are loaded in the webpage, which is very annoying in playlists with hundreds of songs. In order to shuffle the playlist properly, you have to scroll down until the end to load all the songs, then play the first song. Even then, it doesn't feel random enough (works perfectly fine on the mobile app for some reason). So, unlike SoundCloud, ShuffleCloud will make sure that every song in any playlist does not feel left out.
  • It's not working!
    • Check the client ID in the settings and make sure it's 32 characters long. It looks like a bunch of random letters and numbers. This is absolutely necessary in order to access the SoundCloud API.
      • I triple-checked my client ID and it is still giving me errors!
        • There is a rate limit to how many API requests are made from any given client ID (more info). You can see how problems can quickly arise if everyone is using the same ID. This most likely means you have to wait until the 24-hour window expires, or use another client ID (apart from the default).
      • Where can I find a new client ID?
        • Google is your friend, buddy.
    • Make sure you aren't pasting a link of a single track, but a playlist. The link should contain "/sets/" in it.
    • After making sure it's a playlist, double-check that it is set to public. Your private playlists cannot be played for obvious reasons.
    • If you are still having issues with this, please post it on the GitHub repository and I will try to fix it. Be sure to look at the console (Ctrl-Shift-J on Chrome) for any errors.
  • I'm not seeing some of the songs. Why is that?
    • There are a variety of reasons, but the most common one is because the uploader did not want it to be played on third-party apps, or it's simply not streamable. I have not tested tracks that require SoundCloud Go, but I assume that they either won't work or will just stream the first 30 seconds.
